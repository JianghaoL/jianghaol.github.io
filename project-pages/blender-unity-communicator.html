Overview

This is a Blender and Unity add-on that helps developers easily export models to the designated Unity project folder, and also provides a function to overwrite all objects using a specific .FBX model with the newly exported .FBX model.

The tool is designed to streamline the workflow between Blender and Unity, making it more efficient for developers to manage and update their 3D assets.

The application is built using Python for the Blender add-on and C# for the Unity integration.

Implementation

This tool consists of two main components: a Blender add-on written in Python and a Unity script written in C#. The Blender add-on allows users to export their models directly to a specified Unity project folder with a single click.

This was achieved by utilizing Blender's built-in FBX export functionality and integrating it with a user-friendly interface that allows users to set their Unity project path and export settings.

Overwrite Function

One thing that makes this tool stand out is the overwrite function. This function allows users to overwrite all objects in the Unity project that use a specific .FBX model with the newly exported .FBX model from Blender.

Designers are often required to make frequent updates to 3D models during the development process. This function significantly reduces the time and effort needed to update models in Unity, as users no longer need to manually replace each instance of the model.

Specifically, the overwrite function works by searching through all GameObjects in the Unity project for references to the old .FBX model and replacing them with references to the newly exported model.

This is done using Unity's AssetDatabase and GameObject APIs, allowing for efficient and automated updates to the project.

UDP Connection

To facilitate communication between Blender and Unity, I implemented a UDP connection that allows the two applications to exchange data in real-time.

This connection enables Blender to send export commands to Unity, and Unity to receive notifications when new models are available.

A json structure is used to format the data being sent over the UDP connection, ensuring that both applications can easily parse and understand the information.
